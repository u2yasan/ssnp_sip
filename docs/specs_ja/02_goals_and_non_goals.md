# 02. 目的と非目的

## 目的
- voting node の継続性をオペレーターとコミュニティに可視化する
- そのノードが現在 voting 参加可能かを検証する
- 到達性、チェーン追従、finalization 追従を監視する
- 将来の報酬配分に使える登録基盤を作る
- voting key 失効や放置運用による静かな脱落を減らす
- 同一オペレーターや運用代行による集中を抑える

## 非目的
- Symbol のコンセンサスルール変更
- 既存ハーベスト報酬の削減
- トランザクション手数料経済の変更
- CPU ベンチマーク順位表の作成
- ICMP ping を主要スコアにすること
- 単一点監視で合否を決めること
- 品質を問わず全 voting node に均等配分すること
- プロトコルレベルの強制執行やオンチェーン制裁

## 明示的に除外する評価方式
- 外部検証のない自己申告 CPU ベンチマーク
- ICMP ping を主指標にする評価
- 単一リージョン単一点の pass/fail 判定
- finalization や sync 証拠を伴わない「なんとなく速い」評価

## 設計ルール
現在の voting 継続性への信頼を高めない指標は、
中核的な qualification 経路に入れるべきではない。
