# 06. 報酬モデル

## 設計境界
監視・登録基盤は、報酬が未有効でも成立していなければならない。
報酬設計は qualification の下流であり、その逆ではない。

## 資金原則
初期 SSNP は、独立した外部資金または別途承認されたプログラム予算を使う。

## 強い制約
初期設計は以下を削減してはならない。
- 既存ハーベスト報酬
- トランザクション手数料
- ブロック報酬

## 日次プール
R_daily = 名目上の日次プール

## Participation Adjustment
N = Qualified node 数

P(N):
- 1–9 -> 30%
- 10–19 -> 50%
- 20–29 -> 70%
- 30–39 -> 85%
- 40–49 -> 95%
- 50+ -> 100%

R_dist = R_daily * P(N)

未配分分は reserve に残す。

## 順位帯配分
- Rank 1–5: 25%
- Rank 6–10: 20%
- Rank 11–20: 25%
- Rank 21–35: 18%
- Rank 36–50: 12%

## 報酬ロジック
- Qualified node のみがランキング集合に入る
- 最終的な報酬適格判定の前に anti-concentration filter を適用する
- 全 voting node への均等配分は意図的に採用しない

## 注意
資金源の議論が protocol economics に触れ始めた時点で、
それは単なる報酬パラメータ変更ではない。
ガバナンス境界の問題になる。
