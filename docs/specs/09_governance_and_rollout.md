# 09. Governance and Rollout

## Positioning
SSNP should be introduced as a network-stability support program,
not as a protocol rewrite.

## Deployment Phases
### Phase 1
Monitoring, registration, public ranking, and alerts only.

### Phase 2
Manual or semi-manual reward distribution from an external pool.

### Phase 3
Automated reward calculation and distribution, subject to governance approval.

## Governance Principle
Any future move toward protocol-level economic integration
must be treated as a separate governance discussion.

## Communication Rule
Always frame SSNP as:
- not a harvesting reduction;
- not a protocol change;
- not a CPU-performance contest;
- not a popularity reward system.

## MVP Position
The first release should be valuable even if no reward source is approved.
If the monitoring and registration layer is useless without rewards,
the design is wrong.
